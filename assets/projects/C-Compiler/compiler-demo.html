💡 How the Compiler Works: Your C code is tokenized into meaningful units → parsed into an Abstract Syntax Tree (AST) → converted to LLVM Intermediate Representation (IR) → analyzed for pointer aliases and optimizations

Compiler Pipeline

📝 Lexical Analysis
Tokenizes C source code into a stream of tokens. Handles keywords, identifiers, operators and literals with precise position tracking.
🌳 Syntax Analysis
Parses tokens using recursive descent parser to build an Abstract Syntax Tree (AST) representing the program structure.
✔️ Semantic Analysis
Type checking, variable scoping and symbol resolution to ensure semantic correctness of the program.
⚙️ LLVM Code Generation
Translates the validated AST into LLVM Intermediate Representation (IR) for platform-independent code.
🔗 Alias Analysis
Performs flow-sensitive, points-to based analysis to determine memory access relationships and pointer targets.
⚡ SCCP Optimization
Sparse Conditional Constant Propagation eliminates dead code and substitutes constants for improved execution efficiency.
